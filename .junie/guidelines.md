# Проектные гайдлайны и правила для AI Chat

Этот файл описывает архитектурные стандарты, стек технологий и правила написания кода для нашего Unity-проекта. Используй их при генерации и рефакторинге кода.

## 1. Архитектура и Управление Зависимостями
* **Внедрение зависимостей (DI):** Мы используем **VContainer**.
    * Избегай использования синглтонов (`Instance`) и `FindObjectOfType`.
    * Все зависимости должны внедряться через конструкторы (`[Inject]` над конструктором) или через `[Inject]` свойства/поля для `MonoBehaviour`.
    * При создании новых сервисов или систем учитывай их регистрацию в `LifetimeScope`.
* **Событийно-ориентированная архитектура:** Для обмена сообщениями между независимыми системами используй **MessagePipe**.
    * Используй `IPublisher<T>` и `ISubscriber<T>` для слабосвязанных событий.

## 2. Асинхронность и Работа с Данными
* **Асинхронное программирование:** Мы **не** используем стандартные `Coroutines` или `System.Threading.Tasks.Task`.
    * Для всех асинхронных операций (загрузка, таймеры, ожидания, UI-анимации) строго используй **UniTask** (`async UniTask` или `async UniTaskVoid`).
    * Всегда передавай и обрабатывай `CancellationToken` для предотвращения утечек памяти при уничтожении объектов.
* **Конфигурация и статические данные:** Используй **ScriptableObjects** для хранения настроек, игрового баланса и конфигураций данных. Код должен быть спроектирован так, чтобы эти данные было удобно настраивать из инспектора Unity.

## 3. Специфика Unity и Оптимизация
* **GC (Garbage Collection):** Избегай аллокаций в часто вызываемых методах (`Update`, `FixedUpdate`).
    * Предпочитай non-alloc альтернативы физики (например, `Physics.OverlapSphereNonAlloc`).
    * Избегай частого использования строк, конкатенации в `Update` и кэшируй компоненты (`GetComponent`) в `Awake`/`Start`.
* **UI:** При написании логики интерфейсов разделяй UI-представление (View) и бизнес-логику (например, паттерны MVP или MVVM).

## 4. Стиль Кода (Code Style)
* **Именование:**
    * Классы, методы, свойства — `PascalCase`.
    * Приватные поля класса — `_camelCase` с нижним подчеркиванием.
    * Локальные переменные и параметры методов — `camelCase`.
* **Модификаторы доступа:** Всегда явно указывай модификаторы доступа (`private`, `public`, `protected`).
